Attention Mechanisms

In recent years, attention model has been broadly applied to natural language processing, image classification, speech recognition and other different kinds of deep learning tasks.

Human’s attention

From the naming of attention model, it is obvious that it draws on the attention mechanism of human beings. The visual attention mechanism is a brain signal processing mechanism unique to human vision. By quickly scanning the global image, human vision obtains the target area that needs to be focused on, which is the focus of attention, and then invests more attention resources in this area to obtain more detailed information about the target. Other useless information are thus suppressed.

Attention Model

Let’s take machine translation as an example to explain the basic principles of the most common Soft Attention model. After that we try to abstract the essence of the attention mechanism which is not necessarily restricted to the Encoder-Decoder framework. Finally, we give a brief introduction to the recently widely used Self Attention model.

Soft Attention:

When applying Encoder-Decoder framework to machine translation task without the attention model. It’s easy to discover that all words in the target are generated from the same semantic representation $\textbf{C}$.

If the $Source$ sentence is quite long, all the semantic information are completely represented by an intermediate semantic vector $\textbf{C}$, the information of the word itself will be ignored to some extent.

But if we introduce attention mechanism into the translation task, the original intermediate semantic representation $\textbf{C}$ will be replaced with $\textbf{C}_i$ when generating the word $\textbf{y}_i$.

Different $\textbf{C}_i$ may have various attention distribution probability of the Source words:

So, here comes the problem. How do we get the attention distribution probability of the Source sentence for every Target word?

To generate word $\textbf{y}_i$, we can get the value of hidden node in Decoder at $i-1$, let’s denote as $H_{i-1}$. Then we compare the value $H_{i-1}$ with each value of hidden node in Encoder, denote as $h_j$. We do this by calculating the value of the function $F(h_j,H_{i-1})$ which indicates alignment between targe $\textbf{y}_j$ and every Source word.

Attention Essence

Assuming that Source consists of a series of $<\text{Key}, \text{Value}>​$ pair. Then given a Query from Target, our mission is to get the weight of each Value in $<\text{Key}, \text{Value}>​$ by calculating the similarity between Query and Key. Then we add up the Value by weight to get the final Attention.

In stage one, the most commonly used similarity metric are:

In stage two, a Softmax function is applied to get normalized weight from the similarity. And in stage three, Values are added up by its weight.

Self Attention

For normal attention mechanism application in Encoder-Decoder framework, The Source and Target are different. For example, in machine translation task, Source and Target indicate the Chinese and English sentence respectively.

But for self attention, as is like its name, whose attention mechanism happens inside the Source or the Target. Or you can just understand it as Source == Target.

Now the question is: What did self attention learn or what features did self attention extract from “Self”?.
